1. JDK vs JRE vs JVM

1.1 java编译运行过程

java文件(.java) –compile(javac)–> 字节码文件(.class) –execute(java)–> JVM + OS

JVM is a software installed in OS, providing running environment for .class files.

不同OS有不同的JVM, 又java官方提供，如JVM for Linux, JVM for Windows

  • JVM: Java Virtual Machine
  • JRE: Java Runtime Envirment
  • JDK: Java Development Kit

JRE = JVM + Java libraries, 是运行Java的最小环境

JDK = JRE + compile tools & run tool & other develop tools

4. Method

4.1 Signature

Signature = Method name + the number, type and order of its parameters

一个类中不可有两个method签名相同

4.2 Overload vs Override

Overloading occurs in one class when two or more methods have the same method name but different parameters. 编译时绑定

Overriding occurs in two classes, meanning having two methods with the same method name and parameters (i.e., method signature). One of the methods is in the parent class and the other is in the child class. 运行时根据对象的类型(而不是引用的类型)调用

4.3 Constructor

  • 必须和类名相同
  • 无返回值
  • 默认有无参
  • 如果自定义有参Constructor，无参的会消失

5. Memory

JVM内存: 栈 Stack + 堆 Heap + 方法区

  • Heap: new创建的Object

    • 成员变量，类实例化后存储在堆里，随对象一起被回收
    • 访问对象需要引用变量，当一个对象无引用时，被废弃
    • GC (Garbage Collection) 是JVM自带的线程，回收废弃的对象

  • Stack: local variables

    • 方法被调用时放在栈里，随方法调用结束，从栈中清除
	Cell c = new Cell()
	Cell() 
	// 在堆里， 而c在栈里(c是地址)
  • 方法区：存放类的信息
    • 类的类型
    • 类内的方法定义
    • 静态成员变量

7. OOP concepts

  • Encapsulation

    It hides the internal state of one object from the others. It conceals the details.

  • Polymorphism

    Polymorphism is the ability of one object to be treated and used like another object.

  • Inheritance

    Inheritance is an “is-a” relation, which inherits the attributes and behaviors from its parent class.

10. Abstract & interface

  • Abstract Method: 有abstract修饰，无方法体
  • Abstract class:
    • Contains abstract methods 或有abstract修饰
    • Sub-class: 必须实现抽象类的方法，否则也是抽象类
    • 抽象类不可实例化

意义:

  • 提供公共类型
  • 封装子类的重复内容
  • 定义抽象方法

接口 Interface：

  • 特殊的抽象类，只包含抽象方法 (可以有static方法，default方法)，由interface修饰
  • 不可定义成员变量，但可以定义static
  • default method: New feature in Java 8

    Java 8 has introduced the concept of default methods which allow the interfaces to have methods with implementation without affecting the classes that implement the interface.

  • 实现接口时，必须实现所有方法, default methods可以不用

多态 Polymorphism：一个类型的reference，指向不同的object，会有不同的实现

强制转型 Casting：

  • 父类变量 -> 子类变量：object是子类
  • 变量 -> 接口：object实现了接口
  • 用instanceof判断可不可以

内部类：

  • 可以调用外部类的所有成员
  • 对外不可见

匿名内部类：

  • 实现某些接口或继承父类，仅用一次
  • 不需要用class定义
	Runnable run = new Runnable() {
		//实现Runnable接口中的方法
	}

19. Process & Thread

19.1 Introduce

Process: A task in OS. An app runs in a process.

  • 包含一块独立的内存区域，分为一些功能单元。
  • 包含一个或多个线程。拥有一个虚拟地址空间，可被自己的线程访问。
  • 创建时，自动申请一个主线程。

Thread:

  • 进程的顺序执行流。
  • 同类线程共享一块内存和一组系统资源。
  • 包括一个供程序执行时的堆栈。
  • 切换时比进程负荷小。

并发原理：
thread并不是同时执行的，而是有不同的时间段，走走停停。

synchronized锁住的是代码还是对象?答案是：synchronized锁住的是括号里的对象，而不是代码。对于非static的synchronized方法，锁的就是对象本身也就是this。
